### 5. `/coupon` — native form + Patch Retention API (PARTIALLY BLOCKED — see Open Threads)

- Replaced legacy iframe (Patch's hosted form) with a brand-styled native form.
- **Architecture:** browser `/coupon` form → `POST /api/coupon-signup` (our Astro server endpoint, `prerender = false`) → `PATCH api.patchretention.com/v2/contacts?match:phone={phone}` (upsert pattern).
- **Auth:** TWO headers required (caught via bundle scan): `Authorization: Bearer ${PATCH_API_KEY}` + `X-Account-Id: ${PATCH_ACCOUNT_ID}`. Both server-side only.
- **Env vars set on Vercel:** `PATCH_API_KEY`, `PATCH_ACCOUNT_ID`. Until set, page returns friendly 503 with phone fallback.
- **Birthday handling:** form asks for MM/DD only (year irrelevant for birthday-month/day offers); server appends a placeholder year (`1900`) before sending so Patch's date field accepts it.
- **Voice reframe:** `$10 OFF Open Bowling Certificate!` → `$10 toward your next lane.`
- **Birthday input was rebuilt** from native `<select>` dropdowns to a single MM/DD text input — `<select>` open-state can't be styled cross-browser.
- **Trigger mechanism is OPEN.** Earlier broken `POST /v2/tags` step removed (that endpoint creates tag definitions, not contact-tag associations — there's no "apply tag to contact" endpoint in the v2 API). Submissions land in Patch's contact list; the coupon SMS path needs a trigger configured. See Open Threads.

### 6. DNS migration runbook (`Context/dns-migration.md`)

- Captured the full DNS audit for `twistedpin.com`: NS, MX, SPF, DKIM, DMARC, Freshdesk CNAMEs.
- **Flagged: SPF record looks malformed** — `include.com.spf.auto.dnssmarthost.net` is missing the colon after `include`. May be causing silent permerror → DMARC quarantine. Verify with `mxtoolbox.com SPF Check` before cutover.
- Cutover plan: own Cloudflare or GoDaddy DNS → recreate zone → flip nameservers at GoDaddy registrar → 24-48h propagation → verify. User picked GoDaddy DNS (consolidate at one vendor, free).
- **Trump-card fact:** GoDaddy registrar = ownership = no dev-company cooperation needed. They can be silently bypassed.

### 7. Lighthouse baseline + LCP optimization pass

- **Baseline (PSI, mobile):** Perf 86, A11y 100, BP 100, SEO 100. FCP 1.5s ✓, **LCP 3.9s ❌** (target < 2.5s), TBT 0ms ✓, CLS 0 ✓. Desktop is 100/100/100/100.
- **5-fix optimization pass shipped same day:**
  1. Mobile hero serves 540w on phones ≤480px (saves ~1 MB) — single-line markup change, files were already encoded but never referenced. **Biggest single win.**
  2. Hero poster preload — verified already in place with `fetchpriority="high"`. No change needed.
  3. Latin-only `@fontsource` subsets — CSS file 5x smaller per face, fewer `@font-face` declarations to parse.
  4. Tap wall video gains 540w variants — encoder updated + ran; markup uses `<source media>` to pick on phones.
  5. AVIF added to snap image pipeline — 15-30% smaller than WebP at perceptually-equivalent quality. Live for menu hub thumbnails; will auto-generate for other `<picture>` tags when their source images are run through `build-snap-images.mjs`.
- **Deferred:** CRF tuning (mobile video over-encoding) — 540w switch addresses same concern more cheaply. AVIF for non-menu-hub `<picture>` tags — depends on source images being available locally.
- **Re-test discipline:** wait until real photography + final hero splice land, then re-run PSI once. Three measurements that can't be cleanly compared > one measurement against final state.
- **Full history captured in `Context/perf-history.md`** — append-only log with future-run template at the bottom.
