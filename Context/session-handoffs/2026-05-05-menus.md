## TL;DR

This session connected the venue's POS data to the website. Cocktail menu (GoTab), food menu (GoTab), and tap list (Untappd Business) are now driven by real data and refresh automatically every 24 hours. The menus are exposed at three dedicated `/menu/*` pages (extracted out of `/bar` and `/eat` to keep those editorial-focused). Daily 4am Vercel cron rebuilds the site so menu edits propagate without code pushes.

## What shipped

### 1. GoTab integration (cocktails + food)

- **Auth:** OAuth Client Credentials flow against `https://gotab.io/api/oauth/token`. Token format is non-standard (`token` / `tokenType` instead of `access_token` / `token_type`) — handled in `src/lib/gotab.ts`. 23-hour module-level token cache.
- **Data fetch:** GraphQL via `https://gotab.io/api/graph`. Build-time fetch via `src/lib/gotab-cocktails.ts` (source menu = "View Only Cocktail Menu") and `src/lib/gotab-food.ts` (source menu = "View Only Menu", filtered to food categories).
- **Env vars on Vercel (sensitive):** `GOTAB_CLIENT_ID`, `GOTAB_CLIENT_SECRET`, `GOTAB_LOCATION_ID`.
- **Display:** photo card grid, 1:1 squares, 2/3/4 col responsive, 3-line description clamp, optional "Staff Pick" badge from `go:staff favorite` tag.
- **No prices on the website** (premium positioning, ballpark/movie-theater logic — captive audience pricing not benchmarked publicly).
- **Category names rendered exactly as in GoTab** including the truncated `"Cocktails - Reimagined & Handcrafted Fav"`. GoTab is single source of truth for naming; user opted to leave the truncation rather than override in code.

### 2. Untappd Business integration (tap list)

- **Auth:** HTTP Basic with `email:api_token` base64-encoded. No OAuth dance — the API token is the long-lived credential. `src/lib/untappd.ts` handles auth.
- **Data fetch:** REST via `https://business.untappd.com/api/v1`. Build-time fetch via `src/lib/untappd-taps.ts`. Hierarchy: location → menus → sections → items. Filters out generic Untappd placeholder label images so the visual stays clean.
- **Env vars on Vercel (sensitive):** `UNTAPPD_EMAIL`, `UNTAPPD_API_KEY` (note: `_KEY`, not `_TOKEN` — match the dashboard naming), `UNTAPPD_LOCATION_ID`.
- **Display:** list-led rows (different rhythm from cocktail/food grids — tap lists are scannable text). Each row: small label thumbnail (40px mobile / 48px desktop), Glow tap number, beer name, brewery + location + style metadata, ABV. Sections preserved (Draft Beer / NA Draft / Wine on Tap).
- **Confirmed: 28 taps map exactly** (23 + 1 + 4) — brand thesis lines up with the wall.

### 3. Daily auto-rebuild cron

- **Vercel Cron** in `vercel.json`: `0 9 * * *` (9 UTC = 4–5am Central) hits `/api/cron/rebuild`.
- **Route:** `src/pages/api/cron/rebuild.ts` — validates `Authorization: Bearer <CRON_SECRET>` header (auto-injected by Vercel Cron), then POSTs to `VERCEL_DEPLOY_HOOK_URL` to queue a fresh production build. Build re-fetches GoTab + Untappd and bakes current menu state into static HTML.
- **Env vars on Vercel (sensitive):** `CRON_SECRET`, `VERCEL_DEPLOY_HOOK_URL`.
- **Verified working:** manual trigger via Vercel dashboard succeeded, deploy hook fired, fresh production build went live.
- **Hobby-tier flexible window:** cron fires somewhere in the 9–10 UTC hour. Not a problem for menu data.

### 4. Menu pages extracted out of `/bar` and `/eat`

User's read after seeing the embedded menus on `/bar`: "scrolling forever experience on mobile" and "do they ever even make it past the beer tap list to see cocktails." Right call — `/bar` was ~8000px on mobile (~10 viewport heights). Extracted:

- **`/menu/cocktails`** — full cocktail menu only
- **`/menu/taps`** — full 28-tap list only
- **`/menu/food`** — full food menu only

Each has: typography hero (eyebrow + H1 + 1-sentence SEO sub) + the menu itself + a small SEO copy block (H2 + 1-2 paragraphs of keyword-relevant context, includes the address) + SnapFooter. **No image hero** — would rebuild the scroll problem we just solved, and these are reference pages, not editorial.

`/bar` and `/eat` are now editorial-only:
- `/bar` 782 → 435 lines (lost ~6000px mobile scroll)
- `/eat` 558 → 389 lines (lost ~5000px)
- Editorial cocktail / dish teasers preserved (5 staff favorites previewed in the editorial section)
- CTAs repointed: `View what's on tap` → `/menu/taps`, `View the cocktail menu` → `/menu/cocktails`, `View the menu` → `/menu/food`

### 5. Mobile sticky CTA bar polish

User flagged: floating pill had a 12px breathing space below it where page content was visible — read as "pill mid-page" rather than anchored to the bottom. Two iterations:

1. **First attempt** (gradient backdrop fading from transparent above the pill to dark at viewport bottom) — landed visible blur haze above the pill that obscured page content. Bad.
2. **Final** (`src/components/StickyCTABar.astro`): pill bottom margin tightened from 12px → 6px, and a solid (no-gradient) backdrop strip from viewport bottom up to where the pill starts (z-index 29 vs pill's 30, same blur + Indigo Deep wash as the pill). Pill reads as seated on a continuous base. No haze above.
