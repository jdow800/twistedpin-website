## 7. Gotchas

- **Handoff doc was wrong about Astro init status.** Project was empty (root had only CLAUDE.md, Context/, mockups/, node_modules/ from sharp). Verify before re-initing — `ls package.json astro.config.mjs` should both succeed.
- **`mockups/` folder kept** alongside the live Astro app (not under `_archive/`). Iteration history reference. Don't include in deploy — `.vercelignore` excludes it.
- **`Context/videos/` and `Context/pictures/` are gitignored.** Out-of-band sharing required if a session runs on another machine. Pipeline scripts (`scripts/build-hero-video.sh`, `scripts/build-hero-image.mjs`) read from these paths and will fail without them.
- **ffmpeg lives at a non-PATH WinGet location** (`C:\Users\jdow8\AppData\Local\Microsoft\WinGet\Packages\Gyan.FFmpeg_Microsoft.Winget.Source_8wekyb3d8bbwe\ffmpeg-8.1-full_build\bin\`). `scripts/build-hero-video.sh` exports this path explicitly. Don't assume `ffmpeg` is on PATH from a fresh shell.
- **Vercel CLI uploads don't honor `.gitignore`.** If you ever run `vercel deploy` directly, the `.vercelignore` in repo root excludes `Context/videos/`, `Context/pictures/`, `mockups/`, `node_modules/`, `.astro/`, `dist/`, `.vercel/`. Don't remove it.
- **`gh` CLI auth never succeeded this session.** User used the manual GitHub repo creation path. If you need `gh` later, prompt user to run `gh auth login` themselves; the device-code flow doesn't work cleanly from this harness.
- **CRLF line ending warnings on every git commit** — git auto-converts LF→CRLF on Windows. Harmless; don't try to fix.
- **`npm run build` warns of 3 high severity audit vulnerabilities** in transitive dev deps from the create-astro template. Not runtime; not blocking.
- **Range API for text glyph width** — when verifying headline/subhead overflow, use `document.createRange().selectNodeContents(el).getBoundingClientRect()` not `el.getBoundingClientRect()`. Block bounding boxes are container-width by default; Range gives actual rendered glyph width.
- **The `dot` separator in the subhead** is `<span class="dot">·</span>` (a span, not an inline glyph) so the CSS `margin: 0 0.35em` can apply. Don't replace with bare `·` characters — spacing breaks.
- **`hero-fade` initial state** is `opacity: 0; transform: translateY(12px)` set in CSS. The `prefers-reduced-motion` media query in Hero.astro overrides to `opacity: 1; transform: none` — belt + suspenders against JS not running.

## 8. Working pattern

- **Plan first, code second.** For non-trivial tasks, propose component structure / asset plan / motion application points before writing code. User reviews, then execute.
- **One deliverable at a time.** Stop after each meaningful unit of work for user reaction. Don't batch deliverables.
- **Surface, don't iterate** on observations. After a build, list what's worth reacting to (file size trades, layout edge cases, font fallbacks) without fixing them speculatively. User decides what's worth chasing.
- **Update `CLAUDE.md` Decisions Log per session.** Every locked decision goes in the Decisions Log with date + reason. Tomorrow's session reads it before starting.
- **Don't load full-res photos/videos into context.** Use `sharp`/`ffprobe` for metadata; only load derivatives (≤1600px long side) for visual evaluation.
- **Verify visually.** For UI changes, start the Astro dev server (`mcp__Claude_Preview__preview_start` with `astro-dev` from `.claude/launch.json`), inspect with `preview_inspect` / `preview_eval`, and screenshot via `preview_screenshot`. Don't claim a fix works without seeing the render.
